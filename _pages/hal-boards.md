<p>#pragma once: This <g class="gr_ gr_79 gr-alert gr_gramm gr_inline_cards gr_run_anim Grammar multiReplace" id="79" data-gr-id="79">is make</g> so that the header file is present only once in the code. So if A calls the header file B, <g class="gr_ gr_80 gr-alert gr_tiny gr_gramm gr_inline_cards gr_run_anim Punctuation only-ins replaceWithoutSep" id="80" data-gr-id="80">C</g> and B calls the header file C, A would call C two times. With #pragma once declared inside C, A would only call C once. This is useful as calling multiple times might overwrite values that were changed or if the header file is long, delay processing speed.</p>
<p>#define: The format is as follows: #define name value; where <g class="gr_ gr_83 gr-alert gr_gramm gr_inline_cards gr_run_anim Grammar only-ins doubleReplace replaceWithoutSep" id="83" data-gr-id="83">name</g> is the name of the function and value is what you want to assign the name with. So in #define rectangle 50; We are assigning the word the integer 50 to the word "rectangle". If we were to use "rectangle" <g class="gr_ gr_86 gr-alert gr_gramm gr_inline_cards gr_run_anim Punctuation only-del replaceWithoutSep" id="86" data-gr-id="86">in:</g> int number = rectangle; The integer "number" would be assigned <g class="gr_ gr_71 gr-alert gr_gramm gr_inline_cards gr_run_anim Grammar only-del replaceWithoutSep" id="71" data-gr-id="71">with </g>the value 50.</p>

<p>#ifdef CONFIG_ARCH_BOARD_PX4FMU_V1: #ifdef checks if the subject (CONFIG_ARCH_BOARD_PX4FMU_V1) in question has been defined. If it has, it will run the codes up to an #elif, #else or #endif. If not, it will skip the codes between itself and #elif, #else or #endif and run the codes after #elif or #else. So for this example, if "CONFIG_ARCH_BOARD_PX4FMU_V1" has been defined (#define) then it will run</p>

<p>(An "if" statement runs if the requirements <g class="gr_ gr_91 gr-alert gr_gramm gr_inline_cards gr_run_anim Grammar multiReplace" id="91" data-gr-id="91">is</g> fulfilled. So if the name has been <g class="gr_ gr_94 gr-alert gr_spell gr_inline_cards gr_run_anim ContextualSpelling" id="94" data-gr-id="94">defined than</g> the following code runs. All other codes related to the "if" statement is skipped should one "if" statement runs. So if the subject "CONFIG_ARCH_BOARD_PX4FMU_V1" is defined, then it will skip all the codes ranging from the end of "#define HAL_STORAGE_SIZE 8192" to #endif_of which ends the "if" statement. #elif is the code for "else if", so if the previous  "if" or "<g class="gr_ gr_76 gr-alert gr_spell gr_inline_cards gr_run_anim ContextualSpelling ins-del multiReplace" id="76" data-gr-id="76">elif</g>" statement is ignored than another "if" or "<g class="gr_ gr_77 gr-alert gr_spell gr_inline_cards gr_run_anim ContextualSpelling ins-del multiReplace" id="77" data-gr-id="77">elif</g>" statement runs instead. Should one of these had their requirements fulfilled and ran, they will skip to #endif as well.</p>
<p>#undef HAL_OS_POSIX_IO: This is part of the "if" statement as well. What #undef does is the exact opposite of #ifdef. Meaning if the subject in question (HAL_OS_POSIX_I0) is not defined, then the code following it would run. After doing so, it would skip to "endif" as well. So if all the #if and #elif did not run and the subject is not defined then the codes below would run.</p>
<p>#else: If none of the requirements are fulfilled, return an error.</p>

The two would not influence one another but if there is no #endif to indicate the end of the "if" statement, that means the "if" statement is ongoing. You must end an "if" statement with an #endif.</p>
